# 论文结果总览表：数据来源与缺口说明

基准：**提示词 v4**；目标指标：**Jaccard 相似度、消费者福利相对误差、社会总福利相对误差**；场景 **B** 与 **C**，**12 个 LLM**。

---

## 一、场景 C（12 个模型）——数据完整

**数据文件**：`evaluation_results/summary_report_20260205_205348.csv`（以及各模型的 `scenario_c_common_preferences_{model}_*_detailed.json`）。

**理论值（Config A）**：Social Welfare = 209.444，Consumer Surplus = 76.551。

| 指标 | 是否有数据 | 数据位置与计算方式 |
|------|------------|---------------------|
| **Jaccard 相似度** | ✅ 可从现有结果算出，**无需重跑** | 场景 C 的参与决策等价于“参与集合”（参与=1 的用户集合）。在 `*_detailed.json` 的 **config_B.participation.confusion_matrix** 中已有 TP、FP、FN，参与集合的 Jaccard = **TP / (TP + FP + FN)**（两集皆空时取 1）。直接从现有 detailed JSON 即可算 Jaccard，不需重新跑实验。 |
| **消费者福利相对误差** | ✅ 有 | 取 **Config B**（理性中介 × LLM 消费者）行：列 `Consumer Surplus` 为 LLM 结果，理论值 76.551。相对误差 = \|CS_llm - 76.551\| / 76.551。 |
| **社会总福利相对误差** | ✅ 有 | 同上，取 Config B 的列 `Social Welfare`，理论值 209.444。相对误差 = \|SW_llm - 209.444\| / 209.444。 |

**汇总方式**：在 CSV 中按列 `模型` 筛选，每个模型取 Config=B 的一行，即可得到该模型在 v4 基准下（场景 C 的迭代评估默认设置）的上述三个量（Jaccard 用 Individual Accuracy 或自算参与集合 Jaccard）。

---

## 二、场景 B（v4 基准）——仅部分模型有 v4 数据

**数据文件**：`evaluation_results/prompt_experiments_b/b_v4_{model}_*.json`。

**当前仅有 v4 文件的模型**（共 4 个）：
- gpt-5
- gpt-5.1
- gpt-5.1-2025-11-13
- gpt-5.2

**其余 8 个模型**（deepseek-r1, deepseek-v3.2, deepseek-v3.1, deepseek-v3-0324, qwen-plus, qwen-plus-2025-12-01, qwen3-max, qwen3-max-2026-01-23）在 `prompt_experiments_b` 下**没有** `b_v4_*` 文件，因此**没有以 v4 为基准的场景 B 指标**。

**每个 b_v4_*.json 中可直接或间接得到的指标**：

| 指标 | 是否有数据 | 字段与计算 |
|------|------------|------------|
| **Jaccard 相似度** | ✅ 有 | `metrics.jaccard_similarity_mean` 或 `rounds[0].equilibrium_quality.share_set_similarity`。 |
| **社会总福利相对误差** | ✅ 有 | `ground_truth.welfare` 为理论值，`metrics.welfare_mean` 为 LLM 值；相对误差 = \|welfare_mean - ground_truth.welfare\| / \|ground_truth.welfare\|，或直接用 `metrics.welfare_mae_mean / ground_truth.welfare`。 |
| **消费者福利相对误差** | ✅ 能算，无需重跑 | 场景 B 中社会福利 W = platform_value − total_user_cost，平台利润 profit = platform_value − platform_payment，故 **消费者福利（用户侧剩余）= W − profit** = 平台支付总额 − 用户总隐私成本。GT 与 b_v4 结果中均有 `welfare` 与 `profit`，可先算 consumer_surplus_gt = eq_W − eq_profit、consumer_surplus_llm = welfare_mean − profit_mean，再算相对误差 \|cs_llm − cs_gt\| / (\|cs_gt\| + ε)。 |

## 三、小结与建议

1. **场景 C**  
   - 12 个模型在 `summary_report_20260205_205348.csv` 中都有 Config B 的行。  
   - **消费者福利相对误差、社会总福利相对误差**可直接按上表计算。  
   - **Jaccard**：当前仅有 **Individual Accuracy**；若表格要写“Jaccard”，可在 C 中按参与集合定义 Jaccard 并用同一份参与决策数据计算，或表格中场景 C 列注明“结构一致性：Individual Accuracy”。

2. **场景 B（v4）**  
   - 只有 **4 个模型**（gpt-5, gpt-5.1, gpt-5.1-2025-11-13, gpt-5.2）有 v4 的 Jaccard 与社会福利相对误差。  
   - **消费者福利相对误差**在 B 中未实现，需先定义再补算或标注为“—”。

3. **若要做齐 12 个模型 × 场景 B × v4**  
   - 需对另外 8 个模型用 **v4 提示词**再跑一遍场景 B 的静态博弈，并把结果写成与现有 `b_v4_*.json` 相同结构，即可用同一套字段填表。

4. **建议表头（示例）**  
   - 模型 | 场景B Jaccard (v4) | 场景B 社会福利相对误差 (v4) | 场景B 消费者福利相对误差 (v4) | 场景C 结构一致性 (Individual Acc / Jaccard) | 场景C 消费者福利相对误差 | 场景C 社会总福利相对误差  
   - 场景 B 消费者福利列可暂时留空或写“—”，并在表注中说明场景 B 暂无该定义。
